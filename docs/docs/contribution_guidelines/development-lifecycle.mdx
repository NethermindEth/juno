## Contributions Guidelines

See [Engineering Guidelines](./engineering-guidelines.mdx) for more details

### 1. Create a branch off of `main`

Name the branch with a meaningful name.

### 2. Writing code

- Write the code you need to solve the issues:
  - Ensure you have 100% coverage
  - Ensure you add tests for the feature you add
  - Follow this [commit guideline](https://www.conventionalcommits.org/en/v1.0.0/)
  - Add documentation for code and functionality
  - If is the case, write documentation in for the `gojuno.xyz` page, in the `docs` folder of the repository, follow the [Documentation Guidelines](./contribution-guide.mdx)

### 3. Create a pull request on Github

Push your code to Github and create the pull request, link to the issue you are solving

### 4. Submit for review

- Assign the task to yourself in Github and select the appropriate reviewer
- Request a review

## Adding new Issues

Provide as much details as possible for your [Bug](https://github.com/NethermindEth/juno/issues/new?assignees=&labels=&template=bug_report.md&title=),
[Feature](https://github.com/NethermindEth/juno/issues/new?assignees=&labels=&template=feature_request.md&title=) or
follow our security policy in case you found a [Security Vulnerability](https://github.com/NethermindEth/juno/security/policy).
